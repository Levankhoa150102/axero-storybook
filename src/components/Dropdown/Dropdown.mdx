# Dropdown

A flexible dropdown component for navigation menus in Axero projects. The dropdown uses the design system's **Button component** for the toggle button, ensuring consistent styling and behavior across the application.

## Features

- **Button Component Integration**: Uses the existing Button component for consistent styling
- **Click Outside Detection**: Automatically closes when clicking outside the dropdown
- **Flexible Positioning**: Supports left and right alignment
- **Icon Support**: Menu items can include FontAwesome icons
- **Accessible**: Includes proper ARIA attributes for screen readers
- **Customizable**: Accepts custom CSS classes and click handlers

## Navbar Dropdown

The Navbar dropdown displays a "Browse" button with a caret icon. Click the button to reveal the menu. This variant is designed for use in navigation bars with the `navbar` prop enabled.

## Input-Style Selection Dropdown

A dropdown that looks and behaves like an input field, perfect for form selections like categories, locations, or any predefined options. Built using the **DropdownSelect component** with native HTML select elements for better accessibility and mobile experience.

<Canvas of={DropdownStories.WithSelection} />

**CSS Classes Used:** `.input-wrapper`, `.input-label`, `.input`, `.dropdown-select-input`, `.input-error`, `.input-required`

**Props:** `label="Parent category"`, `placeholder="Select category..."`, `items={selectionMenuItems}`, `required={true}`

### Features:
- **Input Field Appearance**: Uses the same styling as Input components for consistency
- **Native Select Elements**: Built with HTML `<select>` and `<option>` for better accessibility
- **Form Validation**: Includes required field validation with error messages
- **Real-time Feedback**: Shows validation status as users make selections
- **Keyboard Navigation**: Full keyboard support with native select behavior
- **Mobile Optimized**: Native mobile dropdown experience on touch devices
- **Screen Reader Friendly**: Proper ARIA attributes and semantic HTML structure

### Use Cases:
- Category selection in forms
- Location dropdowns
- Status selection
- Any form field requiring predefined options
- Replacement for custom dropdowns in forms
