# Radio

A clean and accessible Radio component for single-selection scenarios. Built with semantic HTML using fieldset, legend, and list elements for optimal accessibility and form integration.

## Features

- **Semantic HTML Structure**: Uses `fieldset`, `legend`, and `ul`/`li` for proper form semantics
- **Rich Options**: Supports labels with optional descriptions for each radio option
- **Clean Styling**: Card-like appearance with rounded corners that match the design system
- **Accessibility**: Full keyboard navigation, screen reader support, and proper ARIA attributes
- **Form Integration**: Works seamlessly with forms and validation
- **Required Fields**: Built-in required field indicator

## Default Radio

Basic Radio component with label and description for each option, perfect for the "Type of space" use case.

**Use Cases:**

- Simple category selections
- Yes/No type questions
- Basic configuration options

## Design Patterns

### Option Structure
Each option in the `options` array should include:
- **`value`** (required): Unique identifier for the option
- **`label`** (required): Display text for the option
- **`description`** (optional): Additional descriptive text

### Form Integration
The Radio component works seamlessly with HTML forms:
- Uses proper `name` attributes for form submission
- Supports `required` validation
- Compatible with form libraries like Formik, React Hook Form
- Provides controlled and uncontrolled component patterns
